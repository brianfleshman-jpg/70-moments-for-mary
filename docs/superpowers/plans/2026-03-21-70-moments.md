# 70 Moments for Mary — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a submission form and display site (phone keepsake + TV slideshow) for Mary's 70th birthday, hosted on GitHub Pages.

**Architecture:** Two static HTML pages — `submit.html` (form with Formspree backend) and `index.html` (display with phone/TV modes). A `moments.json` file drives the display. No build tools, no frameworks — vanilla HTML/CSS/JS.

**Tech Stack:** HTML, CSS, JavaScript, Formspree (form backend), GitHub Pages (hosting)

## Chunk 2: Display Page — Phone Mode

### Task 3: Build the display page with phone mode

**Files:**
- Create: `index.html`

- [ ] **Step 1: Create `index.html` with phone mode**

Single self-contained HTML file. Fetches `moments.json` at load, renders scrollable cards.

Structure:
- Hero: gradient "70" + "Moments for Mary" + "A keepsake of love from the people who know her best."
- Cards section: one card per moment from JSON
  - Photo moments: image + text + name/relationship
  - Text-only moments: large italic quote + name/relationship
  - Video moments: `<video>` element with controls + text + name/relationship
- Footer: "70 Moments for Mary"

Key implementation details:
- `fetch('moments.json')` on DOMContentLoaded
- Sort by `number` field
- Generate cards dynamically from the data
- Lazy load images (`loading="lazy"`)
- Video elements: `controls`, `playsinline`, `preload="metadata"`

Styling (matches v2 design doc):
- Font: -apple-system stack
- White background, #1d1d1f text, #86868b secondary
- Cards: #f5f5f7 background, border-radius: 20px, subtle shadow
- Photo in card: full-width, rounded top corners
- Text-only card: centered italic quote, larger font
- Responsive: single column on mobile, 2-column grid on desktop
- Gradient "70" matching submit page

## Chunk 3: Display Page — TV Mode

### Task 4: Add TV slideshow mode to the display page

**Files:**
- Modify: `index.html`

- [ ] **Step 1: Add TV mode to `index.html`**

Detect `?mode=tv` URL parameter. When active, render slideshow instead of scrollable cards.

TV mode implementation:
- Full viewport, dark background (#000), no scrolling, `overflow: hidden`
- One moment visible at a time, crossfade transition between moments
- Auto-advance timer: 12 seconds for photo/text moments
- Video moments: play to completion, then auto-advance
- Progress indicator: "Moment N of 70" top-left, subtle white text at low opacity
- Photo moments: image centered/cover-fit, caption and name at bottom with dark gradient overlay
- Text-only moments: large white italic text centered, name below
- Video moments: video centered, plays with sound
- Moment #70 (last moment): shows "The grandkids — live" holding card, pauses auto-advance
- Keyboard controls:
  - Spacebar: pause/resume auto-advance
  - Right arrow: next moment
  - Left arrow: previous moment
  - F: toggle fullscreen

CSS for TV mode:
- Applied via `.tv-mode` class on body (added by JS when ?mode=tv detected)
- All phone-mode elements hidden
- Slideshow container: `position: fixed; inset: 0;`
- Crossfade: CSS transitions on opacity

Audio unlock:
- On first user interaction (click/tap/keypress), call a silent `audio.play()` to unlock browser audio policy
- Show a subtle "Click anywhere to start" prompt until interaction happens
